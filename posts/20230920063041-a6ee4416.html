Obsidian

这个软件其实相对好了，对于多平台的支持，让我把所有的终端都给安装上了。而且自由度高，主题，插件社区都比较完善。使用这个软件以来，我研究了多端同步和图床，研究了自定义样式，可定制的程度相当高，隐私性也很强。

Obsidian 借助 Remotely save 插件同步坚果云，出现“503，获取元数据错误”，查询网络上说现阶段对于 坚果云 不支持的。

2023 年 10 月 23 日

同步方案太难了，现在最好的应该就是 livesync，但是需要自建 CouchDB，想想 frp 老是断线，还不如用 git 呢。git 想了想，不如用 idea，插件什么的算了。还有个 remoteSync，本来搭配国内坚果云速度和稳定应该都挺好，只是体验上一言难尽，坚果云免费版有限制老是报错，换了个 oneDrive，emmm，可用但不那么理想。而且，这个插件没有看到怎么屏蔽目录，我的 node_modules 都给我上传了干嘛，都是些包文件。

先开始想着来个移动端，PC 端的同步方案，后面发现了移动端续航和同步时效的问题，只选择 PC 之间，结果还是不理想，索性放弃多端同步方案了。

同步

同步工具划分

文件同步借助的坚果云，Gitee，OneDriver。

Gitee 是考虑到某些代码文件需要方便使用，借助的 Git 插件定时同步。
坚果云使用的 PC 工具自动监控同步。
OneDriver 使用的 RemotelySave 插件，手动和定时触发。

同步目录区分

坚果云，Gitee 默认是全量的同步，包括配置、插件、gitee 同步配置等。
OneDriver 非全量，主要是.obsidian 配置和插件目录，还有文档目录同步了，其他的诸如.git,.idea,.cloud 之类的就没有了，插件约束的，具体可见插件的配置面板的描述。.trust 不会被同步到 OneDriver，插件限制的，考虑到 PC 是全量的，主要的操作在 PC，手机主要是看和少量的修改，因此也还行。
值得一提的是，Obsidain 的 PC 插件到了==手机==上还有部分可用，因此手机也是使用的 RemotelySave+OneDriver 同步的。

同步到 OneDriver

借助了插件 RemotelySave，配置好即可，速度不好将就可用。

OneDriver 现在云盘空间为 15G，考虑插件配置占用 1G，其余 10G+ 也够用了。

同步到坚果云

手机端（安卓）的同步不要采用坚果云，实践过，由于坚果云 api 的限制，每次同步到手机就会触发 503 导致失败。电脑端的可以用坚果云。

目录整改方案

最开始是用的坚果云的同步方案，现在想要依靠软件自带的插件来进行同步，也可以支持手机端。

使用的是 CloudFlare 的 R2，现在的问题是插件 Remotely-Sync 的限制上传作用有限（. 和 _），不能把依赖包这种屏蔽掉，因此想了几个方案：

  1. 修改插件，自定义屏蔽规则。
    1. 去 Github 上边下载了源码来看，无从下手；
    2. 考虑到后续更新，非必要不采用这个方案；
  2. 换主题，不需要依赖包那种。
    1. 还没有找到太简约又喜欢的主题；
  3. 分析想要屏蔽的文件夹，主要是 node_modules 和 themes 这两个模块。
    1. node_modules 是 npm 的包依赖，但没有找到可以自定义路径的方案，遂弃；
    2. themes 查找了主题的配置，找到了一个路径，但考虑到 hexo 的隐藏配置有点多，还是决定不采用；
  4. 等待插件支持。
    1. 看到 GitHub 上有人提 Issue ,以后还是有希望解决的；
  5. 隔离 hexo 博客和 obsidian 文件目录。
    1. 因为 hexo 博客主要是依赖 github 和 action 自动构建，只是 md 资源依靠 obisidian 的 md 文件，而 obsidian 只进行文件管理，并不需要对 hexo 的文件进行操作，且隔离开对于 obsidian 的索引更好。
    2. 因此完全可以隔离开，只是 source 目录的同步需要寻找一个方案；
    3. 想过三方的一些同步工具，但其实不想引入太多的工具；
    4. 想到 rclone 的映射，寻找 Windows，找到了类似的方案；

决定使用 5 方案先用着，如果后续 4 方案可行了，再简化。

应该收集什么？

收集箱定位和习惯

在滴答清单的所有智能清单里，只有“收集箱”是没法进行设置的（估计很多酷友都没注意到噢）它作为你下载滴答清单后看到的第一个清单，放在了置顶的位置，其用意也一目了然——那就是官方一直在引导着我们关注“收集箱”。

它作为你工作流程的第一站，收集一切想法，一切念头，一切任务待办，一切待读文章，一切杂七杂八的东西都应该收集在这里。

以我自己为例，收集的东西大致可以分为以下四类：

idea：如怎么实现笔记管理、哪家火锅店好吃、怎么构思滴答清单推文（即一切突如其来的想法和念头）
todo：如忘记取快递了可以今天晚饭过后去取、班群里通知后天下午有班级活动，自己需要腾出时间、有想买的纸质书还没加入购物车（即待办事项和日程安排）
note：如有用的群聊信息、同别人聊天时一些值得记录下的东西、备忘录（即临时的碎片信息）
later：如一些来不及仔细阅读（观看）或者需要反复阅读（观看）的文章/视频的链接（即稍后读/看）

这里可能有小伙伴会问：为什么不用专门的稍后读软件来进行管理？

其实我仔细想了想，原因有二。

一是自己对文章剪藏、防 404 的需求很小很小；二是希望尽可能 all in one。

当然我也不是不使用稍后读软件，如果那篇文章我反复阅读的几率很高的话，那我还是会发送一份到待读 APP 里。（比如“慢读”“cubox”……）

你看，这些最后的归宿都不会是放在滴答清单的收集箱里，而是会被分配到适合它们的地方：或许是成为待办，或许是备忘录，或许是一则笔记，亦或许是一张截图……

而这，就是它作为工作流程第一站的意义。

收集箱任务的初步处理

其实有小伙伴会问，如果这样把各类任务、念头一股脑全往收集箱里塞，不会难以处理吗？对的，当然会。所以其实这一步也相当关键：我们需要对收集箱的任务进行一个初步的处理。

具体步骤其实很简单：

如果这条任务有明确的时间，那么就标上时间。
如果这条任务有明确的属性，那么就标上标签。
这里重点说一下标签。「标签的本质其实是一种“情景”，一种筛选的条件，一个过滤箱。」

收集箱里的任务五花八门、种类繁多，如果不加个标签，你很难选出适合当下的任务。

这里给小伙伴们提供一个设置标签的思路

你可以按照“地点”设置标签，比如某些是只能在外出的情景下完成，有些可以是上课摸鱼完成，有些只能是在图书馆……
你可以按照“工具”设置标签，比如某些任务只能用手机完成，有些只能用平板，有些只能用电脑……
你还可以按照“人物”设置标签，比如某些任务是关联女朋友的，某些任务是关联父母的……

总之，你只需掌握标签的本质作用，剩下的就具体问题具体分析了（小提醒：不要过多设置标签噢）

到期重复 vs 完成重复

自定义重复里边，举个例子解释这两个类型的区别：

假设你原本设置的任务日期是 1 号，每个月重复，但是这个任务你拖延了，3 号才完成，如果设置的到期重复，那么下个任务就还是下个月 1 号，但如果是完成重复，那么下个任务就是下个月 3 号。

什么场景下需要用完成重复呢

比如我想每隔 2 周换床单被套，有时候任务到期了天气不好拖延了 1 周，那么我点击完成后，就是以完成当天再间隔 2 周生成新任务的时间，如果选择普通的到期重复的话，那么才过 1 周就又会提醒我换被套了。

信用卡还款等重复任务

这里要用到任务里设置重复时间的功能，可以按照各种重复频率设置，每次完成后就会自动生成一个新的任务。

我会为每张信用卡建立一个还款重复任务，有点骄傲地说到现在为止四五年没有逾期过。还有交房租、保险扣费、更换隐形眼镜等任务。

除了常规的每日、每周、每月重复，还能设置每个法定工作日、艾宾浩斯记忆法和自定义重复。
自定义重复里面有到期重复和完成重复。举个例子解释这两个类型的区别：

假设你原本设置的任务日期是 1 号，每个月重复，但是这个任务你拖延了，3 号才完成，如果设置的到期重复，那么下个任务就还是下个月 1 号，但如果是完成重复，那么下个任务就是下个月 3 号。

计划和复盘

在我的生活文件夹里第一个任务清单就是计划清单，里面会放我的每日计划、周计划。每日计划不一定每天都会写，但是某天我对下班空闲时间有提前安排的话，就会添加一个每日计划的任务，在详情里把想做的事一件件列好，回家后就可以直接开始做了。

但是我列的这个每日计划和每周计划并不会很刻意去统计完成率，如果我去统计完成情况，一是不容易坚持，二是看到未完成时会自我责备，其实有时候任务没完成是有一定客观原因的，临时有事或者白天工作太忙晚上到家后状态不佳，能做的事就会打折扣，一旦有责备自己的心理，就容易破罐子破摔，索性放飞自我了。

强制必须要完成的事，大多不会拖延，会拖延的大多是一些重要但不紧急的事。虽然很多时间管理的书都会强调我们要把时间花在重要但不紧急的事上，但是作为一个普（fei）通（chai）人，因为缺少足够的内在动力，如果逼迫自己太狠，只会导致过度焦虑，反而适得其反。

计划是为了知道自己还有什么事情可以做，保持进一寸有进一寸的欢喜的心态，只要做好记录，等过一段时间之后复盘，说不定就有惊喜呢。

那么我是如何复盘的呢？很简单，看日历视图。滴答清单的周视图和国誉的手账本很像，有时间轴，每个时间段做了什么都很清晰。

每周我会看一下时间轴，选择标签进行回顾，对自己这一周是怎么过得就心中有数了。
每个月会根据各个清单里面的任务整理一份每月复盘，如果平常不做记录，每月复盘的时候就很容易忘记自己这个月做了什么。

人际关系管理

自从去年更新了多级子任务后，我发现这个新功能可以很好的满足人际关系管理的需求。

用家人朋友的名字建立一级任务，每次和朋友约饭见面都可以在对应的任务里建立一条新的子任务，点开一级任务就可以很清晰地看到上次和这位朋友见面是什么时候。

有必要的话，还可以在朋友的子任务里建立生日的重复任务，这样就不会忘记朋友的生日啦！

除了当天的提醒，一个任务是可以设置多次提醒的，可以在提前一周或者一个月的时候再提醒一次留出时间准备生日礼物。

优先级习惯

我现在的方法是设置高中低优先级，然后在日历视图中按高中低优先级来显示颜色。高表示必须完成，中表示优先完成，低和无优先级表示弹性任务
