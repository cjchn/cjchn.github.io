坚果云

⚠

文件上传大小限制：当前 WebDAV 客户端和网页端上传大小的限制是一致的，默认为 500M。PC 客户端单个文件最大 5G。云盘总容量无限。
访问频率限制：由于 WebDAV 协议比较占用系统资源，免费版用户限制访问频率为每 30 分钟不超过 600 次请求。付费用户限制访问频率为每 30 分钟不超过 1500 次请求。
同步目录数限制：目前坚果云的 WebDAV 协议单次请求文件数（包含文件和文件夹）为 750 个，支持分多页多次加载。
版本记录：免费账户，保存一个月。付费账户，保存三个月。
流量限制：免费上传 1G/月，下载 3G/月。

更多可查看 坚果云-非会员限制

wps 网盘

⚠

wps 上传文件大小限制多少

  • 非会员用户每次能够上传的文件最大不能超过 10M，而所有的文件加起来不可以超过 1G，因为普通用户的云空间是 1G 的大小，里面只能放 1G 以下的内容。
  • 如果是 wps 会员用户，上传文件时每次最高可以达到 2G，而所有的文件加起来可以达到 100G。

SM.MS 图床

免费版存储 5G 容量，单个文件大小限制 5M，每分钟 20 张，每小时 100 张，每天 200 张，每周 500 张，每月 1000 张，免费全球 CDN （cloudflare）加速，无亚洲节点加速。详见 官网描述

不支持相册，找图片需要用搜索功能。参考 集成配置

  1. 什么 sm.ms 图床

sm.ms（https://sm.ms/）是一个比较知名的老牌的图床网站，支持免费图片外链，上传下载速度都不错。支持原图，支持 HTTPS、生成 HTML、BBCode、Markdown 的分享代码。支持格式包括 JPG、JPEG、GIF、PNG 及 BMP，可以批量上传 10 张图片。

  1. sm.ms 图床有什么用

原来借助的 github 搭建的图床，但是因为政策或者墙的问题，图片无法上传，或者是图片链接解析不了，为了方便日常使用，索性找寻可替代的，使用体验更好的图床。

但是免费图床的弊端一直存在，所以 GitHub 一直存在，免费图床的图片也会定期上传到 GitHub 保存，链接也就修改了。

  1. 怎么使用 sm.ms 图床

注册相关的也就不记录了，这里记录下配置 picGo 的过程。

Telegraph-Image

每天 1000 次上传（如果超过该额度，图片管理后台将无法记录新加载的图片），100,000 次免费读取（图片每加载一次都会占用该额度，开启了缓存则不消耗）， 1,000 次免费删除操作（超过将无法删除图片记录），1,000 次免费列出操作（每打开或刷新一次后台都会占用该额度，超过将禁止后台图片管理）

绝大多数情况下，该免费额度都基本够用，并且可以稍微超出一点，不是已超出就立马停用，且每项额度单独计算，某项操作超出免费额度后只会停用该项操作，不影响其他的功能。

由于图片储存在 telegraph 的服务器上，我们无法删除上传的原始图片，只能通过上述第 6 点的黑名单功能屏蔽图片的加载。

CloudFlare R2

R2

官网不方便删除等操作，可以参考 教程 安装 [S3 Browser](https://cji.lanzouu.com/iWHSY1j71bej 密码:bq6d)

配额限制

  • 每个账户可新建 1000 个存储桶
  • 每个存储桶可存储无限个对象
  • 每个对象大小最大 5TB
  • 单次最大上传大小 5GB，可多次分段上传
  • 每秒 1000 次的 A 类操作
  • 每秒 250 次的 B 类操作

收费价格

  • 存储费用：每个月 10GB 免费额度，超出后每个月每 GB 收取 $0.015 的存储费用
  • A 类操作费用：每个月一百万次免费额度，超出后每百万次收取 $4.50 的操作费用
  • B 类操作费用：每个月一千万次免费额度，超出后每百万次收取 $0.36 的操作费用

简单计算一下，1,000,000/30day/24h/60min = 23.148 次/s ，足够日常使用了

Backblaze

Daily Storage Caps：日常存储容量前 10GB 免费
Daily Download Bandwidth Caps：日常下载带宽前 1GB 免费
Daily Class B Transactions Caps：日常 B 类事务前 2500 次免费（B 类事务包括下载、获取文件）
Daily Class C Transactions Caps：日常 C 类事务前 2500 次免费（C 类事务包括创建存储桶、列举存储桶、列举文件版本、列举 Keys）

将我们自己的域名托管到 Cloudfare，使用 Cloudfare 代理并启用免费的 CDN 服务，再将域名指向自己的 Backblaze 云存储地址，这时候使用自己定义的链接请求就是走的 Cloudfare 而不是 Backblaze 了。

由于 Cloudfare 和 Backblaze 同属带宽联盟成员，它们之间数据传输免费，借由 Cloudfare 向 Backblaze 请求数据就不会消耗 Backblaze 的额度！而且 Cloudfare 的 CDN 服务还能够有效提升访问速度。

PicList 的删除是 hidden 操作，文件存储依旧在，形式是加了个版本覆盖；BlazeB2 项目的删除是直接删文件。
