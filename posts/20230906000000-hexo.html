Front-matter

Front-matter 是 markdown 文件最上方以 — 分隔的区域，用于指定个别档案的变数。

Page Front-matter 用于页面配置
Post Front-matter 用于文章页配置

Page Front-matter

写法 解释
title 【必需】页面标题
date 【必需】页面创建日期。例如：2023-09-08 08:00:00
tags 【必需】标签（不适用于分页）
categories 【必需】分类（不适用于分页）
type 【必需】标签、分类和友情链接三个页面需要配置
permalink 【可选】覆盖文章网址
excerpt 【可选】纯文本页面摘录。使用此插件格式化文本
disableNunjucks 【可选】启用时禁用 Nunjucks 标签"\{\{ }}/{% %}"和标签插件的呈现
lang 【可选】设置语言以覆盖 自动检测。继承自 _config.yml
layout 【可选】布局。默认值：config.default_layout
updated 【可选】页面更新日期
description 【可选】页面描述
keywords 【可选】页面关键字
comments 【可选】显示页面评论模块 (默认 true)
cover 【可选】文章缩略图 (如果没有设置 top_img,文章页顶部将显示缩略图，可设为 false/图片地址/留空)

mathjax 【可选】显示 mathjax(当设置 mathjax 的 per_page: false 时，才需要配置，默认 false)

tag-hide

tag-hide 内的标签外挂 content 内都不建议有 h1 - h6 等标题。因为 Toc 会把隐藏内容标题也显示出来，而且当滚动屏幕时，如果隐藏内容没有显示出来，会导致 Toc 的滚动出现异常。

Github Action 自动发布

使用手动发布需要 hexo + node.js + git 等完整的开发环境，如果在新电脑上做一点修改无疑要很大的维护成本。因此选择使用 Github Action，配置好 CI 后，直接用浏览器访问 GitHub 上的项目仓库，直接编辑带错别字的 md 文章，改完，在线提交，稍等片刻，你的网站就自动更新了。

如果手动部署，需要先执行 hexo g 编译生成静态文件， 然后推送 public 整个文件夹到 GitHub 上，当后期网站文章、图片较多时候，很多时候连接 GitHub 不是那么顺畅，经常要傻等很长的上传时间。使用 GitHub Actions 自动部署，你只需 push _post 文件里单独的 md 文件即可，其他不用管，效率瞬间高了许多，其中的好处，谁用谁知道。

使用 GitHub Actions，你还可以一次性将这些静态博客页面部署到多个服务器上，例如：GitHub Pages、Gitee pages、七牛云、阿里云、腾讯云等等。

生成密钥和使用密钥参考 | 脚本参考

主要流程

  1. 生成密钥，并配置上仓库，备用
  2. 除了主目录的 .git，其他目录的全部不要提交到仓库
  3. 配置 deploy.yml

    使用 Github Action 的思路和本地发布差不多，只是是在 Github 提供的一个临时空间操作的。因此需要选择系统版本，安装 Node，Hexo；再是 Hexo 依赖，博客项目依赖；配置 Git，本地仓库，目标仓库；安全考虑需要用公钥密钥。详细的脚本可以参考别人的。

新电脑配置博客

文件的迁移最好是对根路径 hexo_blog 打包（文件夹映射关系需要在 source 不存在时处理），如果不能就先 git 下载，再配置好同步插件，从 R2 同步，这些就弄好了再安装 hexo 环境，ob 插件，测试本地启动。

如果不需要本地启动就不用安装 hexo 环境了，如果不用 ob 管理也不用安装 ob 了，最简单的就是写个 md 直接打开 Github 网页上传，其他的就交给 action 部署了。

以下是配置 hexo 环境的内容：

假设环境已经有了 npm，git

npm 安装 Hexo 环境 npm install -g hexo-cli

拉库 npm i

安装服务工具 npm install hexo-deployer-git --save

启动 hexo cl && hexo generate && hexo douban && hexo algolia && hexo server

命令解释

hexo clean | hexo cl：删除之前生成的文件，若未生成过静态文件，可忽略此命令。如果有搜索，需要执行。

hexo generate | hexo g：生成静态文章

hexo douban | hexo d：生成电影页面

hexo book | hexo b：生成书单页面

hexo game | hexo g：生成游戏页面

以上 3 个的生成命令 hexo -bgm

hexo deploy | hexo d：部署文章，可以用 hexo d 缩写

需要注意的是，通常大家都喜欢用 hexo d 来作为 hexo deploy 命令的简化，但是当安装了 hexo douban 之后，就不能用 hexo d 了，因为 hexo douban 跟 hexo deploy 的前缀都是 hexo d ，你以后执行的 hexo d 将不再是 Hexo 页面的生成，而是豆瓣页面的生成。

hexo algolia: 创建索引，之前需要 clean

hexo server | hexo s | hexo s -p 8000: 本地启动服务

步骤 hexo cl && hexo generate && hexo douban && hexo algolia && hexo deploy

hexo g 生成的静态资源在 .\public 目录

创建新 post 文件 hexo new post 文件名

WARN No layout: index.html 插件错误

删掉 node_module （已经改为不提交该文件夹，因此不用删除了）重新拉取依赖，没能解决。发现 theme 下面是空的，重新安装主题（ \hexo-blog\my-blog 目录下执行）
git clone https://github.com/jerryc127/hexo-theme-butterfly.git themes/butterfly
主题插件 npm install hexo-renderer-pug hexo-renderer-stylus --save
成功
