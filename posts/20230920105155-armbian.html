前言

关于 armbian 的包，找到了Github仓库可以备用。v2ex有个帖子说品控不太好，但是我这个暂时还没有发现。

安装docker

  1. 进去 Armbian系统后输入apt install docker.io 按提示输入y，等待安装完成即可
  2. 输入systemctl status docker 判断docker是否正常运行（绿色active表示正常）

切换到国内镜像库

DDNS-GO

教程参考 | 仓库

注意令牌跟着教程走，不要改其他东西；docker 容器的网络使用 host；注意要填写 ipv4 的 domain，很容易忘记，不然不会执行更新操作。

失败在了光猫没有密码，无法解决端口映射的问题。有想过重置光猫，但是百度了太多失败案例，考虑到没有太多成功方案且失败居多，还是放弃此方案了。

原理其实很简单，就是内网服务器本地配置了一个监控，定期获取公网ip，如果发现和域名的不一致，那么就触发 DNS 服务商（这里测试的 CloudFlare）的更新命令（一个接口），然后域名会更新解析到最新的公网ip，实现动态 DDNS 解析。如果有固定的公网 ip 就不需要这个了，但是因为公网 ip 的缺乏，大多数人都没有固定的，基本上都是由运营商动态变化的。而这个工具也不是万能的，在获取不到或者没有公网 ip 的情况下，无能为力；且这个工具本质只是为了减少切换解析到工作量，如果闲的，也可以不使用该工具，自行去 DNS 服务商手动更新 ip。
